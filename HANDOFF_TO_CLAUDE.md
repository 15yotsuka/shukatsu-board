# Claude Codeへの引き継ぎ資料 (Handoff to Claude Code)

このドキュメントは、ShukatsuBoardの実戦型リニューアル（UI/UXの大規模改修）に関する現在の進捗、変更内容、および今後の残タスクをまとめたものです。Claude Codeはこの内容を完全に把握した上で、即座に実装を継続してください。

---

## 🚀 プロジェクトの目標
50社並行の就活生が実戦で使えるレベルのUI/UXと機能を持つアプリへの大規模改修。
- 「安っぽさ」を排除し、情報密度と操作性を最優先。
- ライト/ダークモード完全対応。
- 無駄な励ましメッセージなどを排除し、実用的なテキストのみ配置。

---

## ✅ ここまでに完了した作業（Done）

1. **ダークモードの完全自律適用**
   - Tailwind v4における `dark:` バリアントの不具合を修正（`globals.css` に `@custom-variant dark (&:where(.dark, .dark *));` を追加）。
   - 全コンポーネント（カレンダー、BottomNav、TrackTabsなど）からハードコードされた白色を排除し、`bg-card` や `dark:bg-zinc-900` に置換完了。

2. **新規パッケージの導入**
   - `canvas-confetti` (紙吹雪エフェクト用)
   - `embla-carousel-react` (ホーム画面カルーセル用)
   - `clsx`, `tailwind-merge`

3. **会社詳細モーダル（1画面集約）の拡張**
   - `types.ts`: `Company` 型に `jobType`（職種）を追加。
   - `CompanyDetailModal.tsx`:
     - 基本情報に「職種」の入力欄を追加。
     - モーダル最下部に、広大な「選考メモ」用テキストエリア（ES回答・面接内容・逆質問などの自由記述欄）を追加実装済み。

4. **モックデータ・ユーティリティの準備**
   - `lib/companyData.ts`: 予測変換用の有名企業一覧（MOCK_COMPANIES）を作成。
   - `lib/confetti.ts`: ステータス変更時の紙吹雪エフェクト関数（`fireConfetti`）を作成。

5. **ナビゲーションの構造変更準備**
   - `BottomNav.tsx`: ボトムナビゲーションを「ホーム(`/`)」「タスク(`/tasks`)」「カレンダー(`/calendar`)」「ES(`/es`)」の4つに拡張済み。

## 🎯 今後やらなければいけないこと（To-Do）

Claude Codeは以下の実装を主導してください（ユーザー承認は取らず、自律的に進めること）。

### 1. ホーム画面 (`/` - ダッシュボード) の実装
- **ヒーローカード**: 「今日やること（直近の面接や締切）」を `embla-carousel-react` で横スクロール表示する。
- **統計タイル**: 「エントリー数」「面接中」「インターン参加」「内定」の4項目のカウントを抽出してグリッド表示（`rounded-2xl` 以上の角丸を適用）。
- ※ 既存のカンバンボードは `/tasks` に移動させるか、この画面からは除外する。

### 2. タスク画面 (`/tasks` - 既存Kanbanの進化/リスト表示)の作成
- `app/tasks/page.tsx` を新規作成する。
- ユーザー指示にある「『企業名 ＋ 選考フェーズバッジ ＋ 次の締切』の構成で、**締切が近い順にソートして一覧表示（リスト表示）**」を実装する。
- ※ 既存のカンバンUIと共存させるか、リスト専用にするかは要件とデザインに合わせて調整。

### 3. スマート入力（サジェスト機能）の実装
- `AddCompanyForm.tsx` または `CompanyDetailModal.tsx` の企業名入力時に、`MOCK_COMPANIES` を使ったサジェストドロップダウンUIを追加。手動での入力も可能とする。

### 4. 成功報酬エフェクト (Confetti) の適用
- ステータスが「インターン参加」または「内定」に変更された際（カンバンでのドラッグ＆ドロップ時や、詳細画面からの変更時）に、`import { fireConfetti } from '@/lib/confetti'` を発火させる。
- 感情的なテキストメッセージ（「やりましたね！」等）は表示せず、エフェクトの視覚情報のみとする。

### 5. 品質保証（QA）
- Windows環境でのUTF-8エンコーディングを厳守。
- 実装が完了したら必ず `npm run build` を実行し、ビルドエラーが0であることを確認・修正すること。
- スクリーンショットによる見た目（ダークモード対応・余白・角丸など）の確認。
