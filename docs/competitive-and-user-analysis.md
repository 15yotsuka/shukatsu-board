# 調査1: 日本の競合ツールUI/UX分析

## 1. Lognavi（ログナビ）

| 項目 | 詳細 |
|------|------|
| **メインUI** | リスト型 + 動画フィード型のハイブリッド |
| **カラー** | ブルー系基調、若年層向けカジュアルな雰囲気 |
| **モバイル** | ネイティブアプリ（iOS/Android）、モバイルファースト設計 |
| **ユーザー数** | 約15万人以上（累計24万人）、利用企業約600社 |

**特徴的なUX:**
- 動画自己PR機能（テキストだけでなく動画でアピール）
- 適性検査ベースのAIマッチング（エース社員との性格マッチ）
- 学生限定SNSコミュニティ
- ワンタップWebセミナー参加

**弱点:**
- ネット不安定時に動画再生・アップロード失敗
- スカウト受信数が少ない（平均1.4社）
- アプリの動作不安定・クラッシュ報告あり
- **選考管理に特化した機能は弱い**（あくまでスカウト型マッチングがメイン）

- 口コミサイトでのレビュー0件（ユーザーフィードバック不足）
- サービス終了済み

## 3. ONE CAREER（ワンキャリア）

| 項目 | 詳細 |

| **カラー** | ブルー/ネイビー系、プロフェッショナル |

- **選考管理はワンキャリア内の企業のみ**（他サイト経由は手動管理）

# 調査1-B: 海外の競合ツールUI/UX分析

## 1. Huntr（huntr.co）

| 項目 | 詳細 |
|------|------|
| **メインUI** | **カンバンボード**（Wishlist→Applied→Interview→Offer→Rejected） |
| **カラー** | ブルー〜パープルのグラデーション、モダン・クリーン |
| **モバイル** | ネイティブiOS/Androidアプリ（**不安定、クラッシュ報告**） |
| **ユーザー数** | 400,000+ |
| **料金** | 無料: 100件 / Pro: $26-40/月 |
| **評価** | G2: 4.9/5（1,100+件）、Chrome: 4.9/5 |

**特徴的なUX:**
- **Map View Tool**: 通勤距離を地図上で可視化（**唯一無二**）
- **KPI/分析ダッシュボード**: 応答率・面接変換率・月次トレンド（**業界最強**）
- **カスタムステータス**: パイプラインステージを自由に追加・変更
- Chrome拡張でワンクリック保存 + 自動データ抽出
- AI履歴書・カバーレター生成

**弱点:**
- 機能が多すぎて学習曲線が高い
- モバイルアプリが不安定（D&Dでクラッシュ）
- Autofillの精度が不安定
- Chrome限定（Firefox/Safari非対応）
- 高額（Pro月$40）

## 2. Teal HQ（tealhq.com）

| 項目 | 詳細 |
|------|------|
| **メインUI** | **テーブル/リスト + カンバンのハイブリッド** |
| **カラー** | ティール（#008080）+ 白 + ダークネイビー |
| **モバイル** | **なし**（Webのみ） |

**特徴的なUX:**
- **Networking CRM**: 連絡先管理×求人紐付け（**業界最強**）
- **Resume-JD Match Score**: 履歴書とJDの一致率+キーワード分析
- **興味度の星評価（1-5）**: フィルタリングにも使える

**弱点:**
- ナビゲーションが散らかっている
- 編集プロセスがclunky
- トラッカーの手動更新が面倒
- **ステータスカスタマイズが限定的**

- キャンセル・返金で悪評

# 調査2: ユーザーの生の声

## ペイン（不満・ストレス）

### Pain 1: マイページ・パスワード管理の地獄

> 「就活をしていると、**30社〜50社程度登録することはザラ**で、その一つ一つのパスワードを被らないようにし、なおかつ覚えておくというのは**至難の技**」
> -- ミキワメ

- 企業ごとに異なるID/パスワード形式
- 「登録したマイページがわからなくなった」「パスワード紛失」が頻発

### Pain 2: メール・通知地獄

> 「1サービスにつき毎日20通ほどの関連性の低いメールが届く」
> -- 就活攻略論

> 「電話が毎日のようにかかってくる。就活生からしたら志望企業からの電話だと勘違いする」
> -- 就活会議レビュー

- メール配信停止しても止まらない
- 退会後もメールが来る

### Pain 3: 情報分散・スプレッドシート管理の限界

> 「エントリーした企業の数が増えすぎて管理が追いつかない」「面接日程が重なってしまった」「タスクが散乱して何から手を付ければいいかわからない」
> -- note.com

> 「膨大なデータが積み上がり、エントリー締切を見落としたり、面接のダブルブッキングに気づかなかったりする」
> -- note.com

- マイナビ、リクナビ、カレンダー、手帳、メモ...情報がバラバラ

### Pain 4: 既存アプリの使いにくさ

**就活会議:**
- 「詳細を見たくてエントリーボタンを押したら**即座にエントリーになり**、取り消せない」
- 「落選した企業が選考ステータスの欄に残り続ける」
- 「登録後に不審なメール・スパムが増加した」

**Lognavi:**
- アプリの動作が遅い、クラッシュ
- オファーのミスマッチ（関係ない企業からばかり）

**TOMOLOG:**
- 「一度登録したものの日時など、編集できるようにして欲しい」

### Pain 5: 「めんどくさい」の根本原因

> 「就活の9割以上は入社しない企業のための作業」
> -- 平均応募12.71社、面接7.78社に対し入社は1社

> 「様々な情報が入り乱れるため、それを整理するのにストレスを感じる」
> -- リクペディア

### Pain 6: 応募後の沈黙（Ghosting）-- 海外ユーザー最大のペイン

> 「People aren't broken by rejection -- they're broken by silence.」
> -- 967件のReddit投稿分析

- **72%の求職者**が就職活動がメンタルヘルスに悪影響と回答
- 35%が応募の認知すらされず
- 40%が2次・3次面接後にゴースト

### Pain 7: 有料機能制限への不満（海外）

- Huntr: Pro月$40、Teal+: 週$13
- 求職者は失業中が多く価格に敏感
- **「結局スプレッドシートに戻った」理由のトップ**

### Pain 8: ブラウザ拡張の不安定さ（海外）

> 「Updates often break the extension or slow it, especially on LinkedIn」
> -- Teal Trustpilot

> 「So painfully slow that it ends up wasting more of your time」
> -- Simplify Trustpilot

## 満足点（「これがあるから使い続けてる」）

### 日本

**就活会議:**
- 「ESや体験記、面接内容を知る事ができるのは本当にありがたい」

**TOMOLOG:**
- 「スケジュール機能メインで使うにはシンプルで使いやすい」
- 「パッと見て今やらないといけないことが分かる」
- 「みんなのカレンダー」機能が好評

**Lognavi:**
- Webテスト機能（偏差値が出る）が好評
- 動画で企業を深く知れる

**Notion:**
- 「すべての就活情報を一元化しやすい」
- 「進捗ごとにカンバンで可視化」「カレンダーで一発確認」
- 「スケジュールの抜け・漏れを減らせる」

### 海外

**カンバンの心理的報酬:**
> 「カードを『Applied』から『Interviewing』に動かすだけで、強い達成感とモメンタムを感じる。スプレッドシートにはない体験」
> -- Careerflow Blog

**ワンクリック保存:**
- Chrome拡張でのジョブ保存が一貫して最も称賛される機能

**情報の一元化:**
- 「メール、LinkedIn、各求人サイトに散らばっていた情報が一箇所に」

**フォローアップリマインダー:**
- 自動通知が精神的負担を軽減

## 就活生の行動パターン

### 管理方法の分布

| 管理方法 | 特徴 | ユーザー層 |
|---------|------|----------|
| **Googleスプレッドシート** | 最も普及、手軽 | 全般 |
| **Excel** | ローカル保存、テンプレ豊富 | PCメイン |
| **Notion** | 一元管理、カンバン可能 | ITリテラシー高め（急増中） |
| **手帳（アナログ）** | 面接中もメモ可能 | 併用が多い |
| **専用アプリ** | シンプルな選考管理 | まだ少数派 |

> 手帳で便利だと思った機能1位は「メモ」（69.7%）

### 同時管理社数

| 指標 | 数値 |
|------|------|
| 平均エントリー数 | 約20〜30社 |
| ES提出企業数（最頻） | 10〜19社 |
| サマーインターン10社以上エントリー | 67% |
| **同時に対応可能な限界** | **約5社** |
| 海外平均（内定まで） | 32件応募 |
| 海外週間ペース | 16件/週 |
| 面接獲得率 | 約6件応募で1回 |

### 管理を始めるタイミング
- 大学2年冬までに約2割がすでに開始
- 本格化: **大学3年の夏インターン**から
- サマーインターン後にNotion等に移行する学生が多い

### カオスになる時期

**ピーク: 大学3年3月〜大学4年4月**

| 時期 | 状況 |
|------|------|
| **3月** | 大手一斉公開、説明会・エントリー受付開始、ES提出ピーク（月平均4.6社） |
| **4月** | 複数社の選考同時進行、面接ピーク |
| **5〜6月** | 面接・内々定ピーク、「持ち駒管理」と「辞退連絡」同時発生 |

- 約4割が**時間の把握ミス**を経験（Web予約→手帳転記時のミス）

### 情報の記録場所（分散状態）
- マイナビ/リクナビ: エントリー情報、説明会予約
- Googleカレンダー: 面接日程、ES締切
- スプレッドシート/Excel: 企業リスト、ID/PW、選考状況
- Notion: 企業研究ノート、ES下書き
- 手帳: 面接中メモ、説明会メモ
- Googleドキュメント: ES下書き（コピペ用）
- LINE/メモアプリ: 速記メモ
